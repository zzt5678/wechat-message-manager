# 应急降级附录

Windows 当前版本恢复失败时，首选等待项目适配或手动导入已有 key。临时使用旧版微信会关闭/替换程序、要求重新登录，并带来供应链、数据迁移和自动升级风险，因此不提供一键脚本，也不是教程主流程。

只有在用户明确批准每个状态变化后才考虑：

1. 备份当前程序和本地数据库；确认备份不进入 Git/云盘。
2. 仅使用来源可信、Authenticode 状态为 `Valid`、签名主体为腾讯的安装包。
3. 记录并核验版本和 SHA-256，不从第三方“绿色版”下载。
4. 用户手动退出微信、安装兼容旧版、登录并完成一次只读捕获。
5. 恢复官方当前版本并再次验证数字签名。
6. 删除旧安装包、临时程序副本和备份，重新执行 full refresh 与 7/7 `quick_check`。

本机曾在腾讯签名的 4.1.9.57 完成过这一应急验证，随后恢复到 4.1.11.54；但当前版本直接恢复已经成功，因此新用户不应先走降级路径。
