# 安全模型

## 保护对象

- 数据库 key 与账号口令
- wxid、联系人/群稳定标识
- 原始聊天消息和明文数据库
- 能反推出账号目录的绝对路径

## 信任边界

原始微信数据库始终只读。解密结果、manifest、状态、导出和回执只进入本机私密 vault。Windows key 由当前用户 DPAPI 保护；macOS key 由当前用户 Keychain 保护。删除 vault 可回滚项目状态，但不会修改微信本身。

## 验证链

```text
候选 key/口令
  → 每个核心数据库 page-1 HMAC
  → 解密时每一页 HMAC
  → 源文件前后指纹一致
  → SQLite quick_check
  → 查询前 freshness 指纹一致
```

任一环节失败即停止。工具不使用“能打开”“有几条消息可读”作为成功判断。

## 明确不做

- 不发送消息或自动控制微信。
- 不采集截图、OCR、剪贴板或无障碍树。
- Windows 不注入、不 Hook、不写目标进程。
- 不上传配置、数据库、图片索引或 key 到远端服务。
- 不在日志/回执显示 key、salt、wxid、私密路径或批量正文。

## macOS 例外

macOS 没有经过本项目真机验证的被动 V4 恢复路径。Frida 方案因此被隔离为单独命令，要求再次明确同意，只允许操作副本。若不接受此风险，只使用手动导入。

## Windows 旧版应急例外

默认 Windows 捕获始终只读。只有当前版恢复明确不兼容且用户单独批准时，4.1.9 应急状态机才允许临时改变程序文件。它固定校验腾讯 HTTPS 来源、制品大小、公开 SHA-256、Authenticode 签名和准确版本；切换前备份当前启动器并稳定快照核心加密数据库，保留当前版本目录，捕获后要求恢复和真机运行时复核。

旧版捕获仍然只有 `PROCESS_VM_READ | PROCESS_QUERY_INFORMATION`，不注入、不 Hook、不写进程。项目不静默安装、卸载、启动、退出或登录微信。程序切换、内存读取、恢复和清理各自需要显式同意；任何状态、哈希、签名、HMAC 或路径边界异常都会停止。

## 发布前检查

仓库 CI 会运行单元测试和文本秘密扫描。发布者还应检查 `git status`、`git ls-files` 和仓库对象，确保安装包、备份、vault、审计回执和第三方 vendor 目录从未进入提交历史。
